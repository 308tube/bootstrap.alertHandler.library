Msg Code Library

The Msg Code Library function that removes all the alert messages.
This will be the place where you can define all your positive (example: Good Job!) and negative (example: Error) messages.

In the example below, you will notice that all negative message are represented with a negative number "key". All positive messages are represented with a positive number "key". This is to help developers quickly assess whether the message or a good or bad message. Zero should be left out or used as a "test" message. 

//this is what the library function looks like.
function _bootstrap_alert_msg_library()
{
	//return array of objects
	return [
		{
			key: 0, //key is stored as a numeric message
			display_msg: "This is the message displayed to the users",
			sys_msg: "This is the message that will be sent up to the logging service"
		},
		{
			key: -1, 
			display_msg: "A negative numeric number signify a error message",
			sys_msg: "All negative numbers are error messages. This allows the developer" +
	       			"to quickly understand if a message is good or bad."
		},
		{
			key: 1, 
			display_msg: "A positive numeric number signify a warning or positive message",
			sys_msg: "All positive numbers are for warning and positive messages. This allows " +
	       			"the developer to quickly understand if a message is good or bad."
		}
	];
}

How to add a message to the Library

In this example will want to add to the library above. Lets list out what this is built for.

Business Requirement

  • A error message telling the user you left a input blank
  • Error message is specific to one input (User email address)
  • Message display to user will be "Error please enter you email address"
  • System message to be logged is "User did not enter email address newUser.html"
  • Logs will be track for analysis

From the business requirement we build the simple Technical Requirements

  • Error is a negative message, so the key number is
  • From the business requirements we know what the user message will be. "Error please enter you email address"
  • From the business requirements we know that they want a specific error message but they also want to track these errors. Know this you should tell the business unit that the system message should be formatted in such a way that is easy to parse and statistical analysis on it.

Add the message to the library

//this is what the library function looks like.
function _bootstrap_alert_msg_library()
{
	//return array of objects
	return [
		{
			key: 0, //key is stored as a numeric message
			display_msg: "This is the message displayed to the users",
			sys_msg: "This is the message that will be sent up to the logging service"
		},
		{
			key: -1, 
			display_msg: "A negative numeric number signify a error message",
			sys_msg: "All negative numbers are error messages. This allows the developer" +
	       			"to quickly understand if a message is good or bad."
		},
		{
			key: 1, 
			display_msg: "A positive numeric number signify a warning or positive message",
			sys_msg: "All positive numbers are for warning and positive messages. This allows " +
	       			"the developer to quickly understand if a message is good or bad."
		},
		{
			key: -2, //new error message
			display_msg: "Error please enter you email address", //user message
			sys_msg: "-2;newUser.html;User did not enter email address newUser.html" //sys msg, use ; to parse messages
		}
	];
}
